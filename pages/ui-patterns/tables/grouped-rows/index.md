---
title: Grouped rows
layout: design-pattern
category: Tables
permalink: ui-patterns/tables/grouped-rows/
design-pattern-type: mobile

what: >
 Section headers inside a table that group the data. Example: collections by year or by category.

why: >
 To improve readability of a table, for quickly identifying different groups of data to find a particular cell.

do: >
 * Visually differentiate the summary rows from the other rows in the table.
 
 * The summary rows should be thin and subtle.

dont: >
 * Be inconsistent on alignments, this rests readability.

tags: >
 List, group, items, table, data, sections, categories, rows.
